# 企业内网拓扑文字版

> **注意**: 本文档描述原始设计（VRRP + RIP + OSPF 混合）。实际 eNSP 部署采用**纯 OSPF 方案**（无 VRRP/RIP），IP 地址亦有调整。实际配置见 `ensp_config_commands.txt`。本文档保留作拓扑结构参考。

## 一、拓扑概述

本组网为分层架构企业局域网，分为**出口路由层、核心交换机层、汇聚交换机层、终端接入层**；出口路由器对接互联网，核心双机冗余（VRRP+链路聚合），路由采用OSPF+RIP+静态路由重分布混合部署，实现全内网互通+互联网访问。
分层结构：出口R1→双核心C1（备用）/C2（主用）→汇聚D1/D2→终端计算机/云主机/互联网终端。

## 三、关键组网技术说明

1. **链路聚合**：C1与C2之间采用Channel Group链路聚合，提升链路带宽与可靠性；
2. **VRRP网关冗余**：`10.1.1.16/29`网段部署VRRP，虚拟网关 `10.1.1.22/29`，C2为主网关、C1为备用网关，核心故障自动切换；
3. **路由协议规划**
   - 核心骨干（C1↔C2、核心↔汇聚）：**OSPF，全部运行在Area 0骨干区域**；
   - 出口R1与双核心之间：运行RIP协议（RIP网段 `10.1.1.20/29`）；
   - 核心→汇聚的用户业务网段：使用静态路由，**静态路由重分布导入OSPF进程**，实现内网网段全路由互通；
4. **互联网出口**：R1作为全网出口网关，内网通过R1访问互联网终端。
