## 一、全部完成

### 1. 网络拓扑搭建 ✅
- eNSP 仿真拓扑: R1(出口) → C1/C2(双核心, Eth-Trunk) → D1/D2(汇聚) → PC-A/B/C/D + InterNet1 + Cloud1(VM桥接)
- 路由协议: OSPF Area 0 全网统一
- VM (192.168.179.136) 与拓扑内任意设备互通

### 2. SNMP Agent 配置 ✅
- 五台设备(R1/C1/C2/D1/D2)已开启 SNMP v2c, community `bjfu11mx`
- VM snmpd 已配置 (aux/snmpd.conf): `rocommunity bjfu11mx` + `disk /` + exec 脚本

### 3. BIND DNS ✅
- Ubuntu VM BIND9, 解析域 `example.edu.cn`
- 五条 A 记录: r1/c1/c2/d1/d2.example.edu.cn
- 配置脚本: `bind_setup.sh`

### 4. SNMP 采集程序 (C 语言) ✅

#### 功能特性
- **守护进程**: fork + setsid + PID 文件锁 (`/var/run/snmp_collector.pid`)
- **单实例**: flock 排他锁，二次启动拒绝
- **60s 轮询**: while(1) 主循环
- **纯异步采集**: asynchapp.c 模式，snmp_send + select 事件循环
- **指标**:
  - 网络设备每接口 9 项: sysName + 8 个接口计数器
  - VM 6 项: memTotalReal, memAvailReal, dskTotal, memTotalSwap, dskUsed, named CPU/RSS
- **配置文件驱动**: devices 和 metrics 全部从 collector.cfg 读取

### 5. Web 前端 ✅

#### API (Laminas)
| Route | 说明 |
|---|---|
| `GET /api/topology` | 节点+边 JSON，内嵌 ICMP ping 颜色状态 |
| `GET /api/devices` | 设备 + 最新指标汇总 |
| `GET /api/device/{name}` | 单设备详情 |
| `GET /api/metrics/{name}` | 时序数据（供图表） |

#### 前端页面
- **vis.js 拓扑图**: 12 节点布局固定，无箭头边，60s ICMP 监测刷新颜色
- **EasyUI datagrid**: 设备/指标列表，点击查看详情
- **ECharts**: ifInOctets 折线图 + VM 内存饼图
- nginx 配置: `web/nginx-site.conf`
