# 前端代码阅读顺序

## 阅读顺序

```
module.config.php → ApiController.php → layout.phtml → index.phtml
```

---

## 1. module.config.php — 路由 + 控制器工厂

**路径**: `web/module/Application/config/module.config.php`

| 路由 | 类型 | URL | 对应方法 |
|------|------|-----|----------|
| home | Literal | `/` | `IndexController::indexAction` |
| api-topology | Literal | `/api/topology` | `ApiController::topologyAction` |
| api-devices | Literal | `/api/devices` | `ApiController::devicesAction` |
| api-device | Segment | `/api/device/:name` | `ApiController::deviceAction` |
| api-metrics | Segment | `/api/metrics/:name` | `ApiController::metricsAction` |

关键点：
- `Literal` 是精确匹配，`Segment` 带 `:name` 参数
- `ApiController` 的工厂函数用匿名函数创建 PDO 连接并注入（MySQL host/user/password 来自 `config/autoload/global.php`）
- `ViewJsonStrategy` 让 Action 返回 `JsonModel` 时自动输出 JSON

## 2. ApiController.php — 4 个 JSON API

**路径**: `web/module/Application/src/Controller/ApiController.php`

### 构造函数
- 接收 PDO，存到 `$this->pdo`

### GET /api/topology — 拓扑图数据（行 21-71）
- 硬编码 12 个节点（设备名 + IP + 分组）、12 条边
- Cloud1 的 IP 为 null（不 ping）
- 每个节点调 `pingCheck()` 决定颜色（green/red）
- 返回 `{nodes: [...], edges: [...]}`

### GET /api/devices — 设备列表（行 74-93）
- SQL: 子查询 `MAX(created_at)` + JOIN 自己，取每个 (device, metric) 的最新一条
- `cleanRows()` 去掉 value 的 SNMP 类型前缀（如 `Counter32: 123` → `123`）
- 返回 `{devices: {R1: [...], VM: [...]}}` —— 按设备名分组

### GET /api/device/{name} — 单设备详情（行 96-116）
- 同上 SQL 逻辑，加 WHERE 限定单个设备
- 返回 `{device: "R1", metrics: [...]}`

### GET /api/metrics/{name} — 时序数据（行 119-141）
- 白名单校验（6 种指标），不在名单返回 400
- SQL: `WHERE metric LIKE 'ifInOctets%' ORDER BY created_at DESC LIMIT 300`
- 返回 `{metric: "ifInOctets", data: [...]}`

### 辅助方法
- `cleanRows()` — 遍历行，调 `stripSnmpPrefix()` 去掉 `Counter32:` 等前缀
- `stripSnmpPrefix()` — 找第一个 `:`（位置 < 20），截掉冒号前的前缀
- `pingCheck()` — `exec('ping -c 3 -W 1 IP')`，从输出最后一行解析丢包率

## 3. layout.phtml — 页面框架

**路径**: `web/module/Application/view/layout/layout.phtml`

- `<head>` 引入 EasyUI CSS、vis.js CSS、页面内联样式
- `<body>` 顶部标题栏 + `$this->content`（插入 index.phtml）
- 底部引入 jQuery、EasyUI JS、vis.js standalone UMD、ECharts
- 页面布局：flex 四栏（左上拓扑 50%×50% / 右上表格 50%×50% / 下方图表 100%×50% 左右各半）

## 4. index.phtml — 前端 JS 逻辑

**路径**: `web/module/Application/view/application/index/index.phtml`

### Vis.js 拓扑图（行 15-78）
- 12 个节点固定位置，5 种分组（router/switch/pc/cloud/server）不同颜色形状
- physics 关闭，交互禁用（不可拖拽缩放）
- `refreshTopology()`: 首次加载设位置+固定，之后只更新颜色
- 每 60s 调用一次

### EasyUI DataGrid（行 80-147）
- 7 列：设备名 / 系统名称 / 入流量 / 出流量 / 指标数 / 最近更新 / 详情链接
- `loadFilter` 把 API 返回的 `{devices: {...}}` 转为行数组
- 入/出流量汇总：遍历 metrics，累加所有 `ifInOctets.*` / `ifOutOctets.*`
- `formatTraffic()`: 自动转换为 B/KB/MB/GB
- 详情链接打开 EasyUI dialog，显示该设备的所有 metric

### ECharts 图表（行 149-219）
- **折线图（ifInOctets）**: 从 `/api/metrics/ifInOctets` 取数据，按 `设备.接口` 分组，过滤全零 series，X 轴显示时间
- **饼图（内存）**: 两次 AJAX 取 memTotalReal 和 memAvailReal 的最新值，已用=total-avail，实心饼图
- 每 60s 刷新
- `resize` 事件响应窗口变化
