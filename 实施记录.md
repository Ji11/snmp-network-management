## 一、eNSP 拓扑相关

### 2026-07-06 — 重启后拓扑恢复

- **问题**: 电脑重启后 eNSP 设备配置丢失
- **操作**:
  - R1/C1/C2/D1/D2/PC-A/B/C/D/InterNet1/Cloud1 重新导入 `ensp_config_commands.txt` 全部配置命令
  - C1↔C2 Eth-Trunk 的 OSPF 未激活 → 手动 `int Eth-Trunk1; ospf enable 1 area 0.0.0.0`（后根据新拓扑方案确认 Eth-Trunk 不应在 OSPF Area 0，已回退）
  - C1↔C2 之间两条 GE 线在 eNSP 界面重新连接（GE1/0/2↔GE1/0/2, GE1/0/3↔GE1/0/3）
  - D1 GE0/0/1 重启后物理 DOWN → `undo shutdown` + eNSP 重连网线
  - C1↔D1 Vlanif101 恢复后 10.1.1.6 不响应 ping（eNSP Vlanif 已知 bug），重启拓扑后恢复

### 拓扑架构要点

| 项目 | 值 |
|------|-----|
| 路由协议 | OSPF Area 0（R1↔C1, R1↔C2），C1↔C2 Eth-Trunk **不在** OSPF 内 |
| C1↔C2 互联 | Eth-Trunk1 L3，10.1.1.0/30，纯直连无动态路由 |
| C1↔C2 间路由 | 经 R1 中转：C1 → R1 → C2 |
| VM 接入 | C2 Vlanif10 (192.168.179.1/24) → Cloud1 L2 桥接 → VM ens33 (192.168.179.136/24) |
| SNMP community | `bjfu11mx`，所有设备 v2c |
| eNSP 设备型号 | R1=AR2220, C1/C2=CE12800, D1/D2=S3700, Cloud1=Cloud, 终端=PC |

### 已知缺陷

- D1/D2 (S3700) 部分 VLAN 接口不支持 ifInErrors OID
- C2 (CE12800) Vlanif10 不支持 ifInErrors
- eNSP 重启后 VLAN 接口偶发 ping 不响应，需全拓扑重启

## 三、VM 网络配置

### NetworkManager 持久化路由

```bash
# 查看
nmcli con show "Wired connection 1" | grep ipv4.routes

# 当前配置
ipv4.routes: { ip = 10.0.0.0/8, nh = 192.168.179.1 }
             { ip = 192.168.0.0/16, nh = 192.168.179.1 }
```

- `10.0.0.0/8` — 覆盖 eNSP 内部拓扑所有 10.x.x.x 网段
- `192.168.0.0/16` — 覆盖 D1/D2 下联 PC 网段（192.168.2-5.0/24）

### 添加新路由（永久）

```bash
sudo nmcli con mod "Wired connection 1" +ipv4.routes "<网段> <下一跳>"
sudo nmcli con up "Wired connection 1"
```

### VM SNMP Agent

- 配置: `/etc/snmp/snmpd.conf` — `rocommunity bjfu11mx` + `disk /` + exec 脚本
- named 监控脚本: `/usr/local/bin/named_cpu.sh`, `/usr/local/bin/named_mem.sh`
- 采集器用 extOutput OID (`.1.3.6.1.4.1.2021.8.1.101.1/.2`) 取 named CPU/RSS

## 四、部署清单（新环境从头搭建）

### 1. eNSP 拓扑

在 eNSP 中打开拓扑 → 逐设备粘贴 `ensp_config_commands.txt` 中的配置命令 → Cloud1 绑定 VMnet 网卡 → 启动所有设备。

### 2. VM 依赖

```bash
sudo apt install libconfig-dev libsnmp-dev libmysqlclient-dev build-essential
sudo apt install nginx php8.1-fpm php8.1-mysql php8.1-json composer
sudo apt install bind9 snmpd mysql-server
```

### 3. SNMP Agent (VM)

```bash
sudo cp aux/named_cpu.sh aux/named_mem.sh /usr/local/bin/
sudo chmod +x /usr/local/bin/named_{cpu,mem}.sh
# 编辑 /etc/snmp/snmpd.conf，确保有 rocommunity bjfu11mx + disk / + exec 两行
sudo systemctl restart snmpd
```

### 4. MySQL

```sql
CREATE USER IF NOT EXISTS 'root'@'localhost' IDENTIFIED BY '123456';
GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```
采集程序启动后会自动建库建表，无需手动操作。

### 5. BIND DNS

```bash
sudo cp /etc/bind/db.bjfu.edu.cn /etc/bind/db.example.edu.cn
sudo vim /etc/bind/db.example.edu.cn   # 替换 bjfu → example
sudo vim /etc/bind/named.conf.local    # 追加 example.edu.cn zone
sudo systemctl restart named
```

### 6. 编译 + 启动采集器

```bash
cd ~/NetProgramming/CourseDesign && make clean && make
./build/snmp_collector config/collector.cfg &
# 停止: kill $(cat /tmp/snmp_collector.pid)
```

### 7. Web 前端

```bash
sudo cp web/nginx-site.conf /etc/nginx/sites-available/snmp-monitor
sudo ln -sf /etc/nginx/sites-available/snmp-monitor /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx
# 访问 http://127.0.0.1/
```

### 8. 网络路由（永久）

```bash
sudo nmcli con mod "Wired connection 1" +ipv4.routes "10.0.0.0/8 192.168.179.1"
sudo nmcli con mod "Wired connection 1" +ipv4.routes "192.168.0.0/16 192.168.179.1"
sudo nmcli con up "Wired connection 1"
```
