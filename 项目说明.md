# SNMP 网络管理系统

> 状态: 全部完成 | 更新: 2026-06-06

## 1. 项目概述

基于 libnetsnmp + libconfig + MySQL 的 SNMP 数据采集系统，前端 Laminas (PHP) + vis.js + EasyUI + ECharts。

- 采集 5 台网络设备 + 1 台 VM 的运行指标
- 守护进程模式，60s 轮询间隔，单实例锁
- 异步采集（asynchapp.c 模式）
- Web 前端展示拓扑、数据列表、折线图、饼图
- ICMP ping 60s 监测拓扑节点状态

## 3. 采集指标

### 网络设备（每接口 9 项）
sysName, ifInOctets, ifInUcastPkts, ifInNUcastPkts, ifInErrors, ifOutOctets, ifOutUcastPkts, ifOutNUcastPkts, ifOutErrors

### VM 设备（6 项）
memTotalReal, memAvailReal, dskTotal(/) , memTotalSwap, dskUsed(/), named CPU/RSS (extResult)

## 4. 编译与运行

```bash
# 编译
cd ~/NetProgramming/CourseDesign && make clean && make

# 运行（后台守护进程）
./build/snmp_collector config/collector.cfg &
# 或
./build/snmp_collector config/collector.cfg  # 前台调试

# 停止
kill $(cat /var/run/snmp_collector.pid)
```

## 5. 部署清单

### VM 端
1. SNMP Agent: `sudo cp aux/snmpd.conf /etc/snmp/snmpd.conf`
2. named 脚本: `sudo cp aux/named_cpu.sh aux/named_mem.sh /usr/local/bin/ && sudo chmod +x /usr/local/bin/named_*.sh`
3. 重启: `sudo systemctl restart snmpd`
4. DNS (可选): `bash bind_setup.sh`
5. nginx: `sudo cp web/nginx-site.conf /etc/nginx/sites-available/snmp-monitor && sudo ln -sf ... && sudo systemctl reload nginx`
6. 采集程序: `./build/snmp_collector config/collector.cfg &`

### 已知缺陷（eNSP 模拟设备限制，非代码 bug）

| 设备 | 缺失指标 | 原因 |
|------|----------|------|
| D1 (S3700) | `ifInErrors.6`, `ifInErrors.7` | S3700 Vlanif20/Vlanif30 不支持 ifInErrors |
| D2 (S3700) | `ifInErrors.6`, `ifInErrors.7` | S3700 Vlanif40/Vlanif50 不支持 ifInErrors |
| C2 (CE12800) | `ifInErrors.18` | CE12800 Vlanif10 不支持 ifInErrors |

采集器收到 noSuchName 时正常跳过，不影响其他指标。
